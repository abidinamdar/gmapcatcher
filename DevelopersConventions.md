# Developers conventions #

Developers conventions used in GMapCatcher.

In the file should be rules used by developers. Python coding conventions, standards as well as desired behavior is described in the file. If ones want to make change in behavior of the application and it is described in the file, the change should be discussed before it is implemented to the application.

# Python conventions #

**Using PEP8**

http://www.python.org/dev/peps/pep-0008/

  * **Ignored rules**
    * **E501** - line too long (82 characters)
    * **E123** - closing bracket does not match indentation of opening bracket's line (might change)
    * **E126** - continuation line over-indented for hanging indent
    * **E127** - continuation line over-indented for visual indent
    * **E128** - continuation line under-indented for visual indent
